I18n 사용 규칙

1. 기본 규칙

  • 모든 정적 텍스트는 i18n을 사용하여 다국어 처리할 것
  • 하드코딩된 한글 텍스트 사용 금지
  • translation.json 파일은 직접 수정하지 말고 Google Sheets를 통해 관리할 것

2. 키 네이밍 규칙

  • 키는 계층 구조로 작성하고 점(.)으로 구분
// 올바른 예시 ✅
{
  "common": {
    "title": {
      "welcome": "환영합니다"
    },
    "button": {
      "save": "저장",
      "cancel": "취소"
    }
  }
}

// 잘못된 예시 ❌
{
  "commonTitleWelcome": "환영합니다",
  "commonButtonSave": "저장",
  "commonButtonCancel": "취소"
}

3. 컴포넌트 사용 규칙

  • 컴포넌트에서 i18n 사용시 setup에서 useI18n 훅 사용
<!-- 올바른 예시 ✅ -->
<script setup>
import { useI18n } from '@/_setting/i18n'

const { t } = useI18n()
</script>

<!-- 잘못된 예시 ❌ -->
<script setup>
const messages = {
  ko: { welcome: '환영합니다' }
}
</script>

4. 변수 포함 텍스트 규칙

  • 변수가 포함된 텍스트는 중괄호({}) 구문 사용
<!-- 올바른 예시 ✅ -->
<template>
  <p>{{ $t('user.greeting', { name: userName }) }}</p>
</template>

<!-- 잘못된 예시 ❌ -->
<template>
  <p>안녕하세요, {{ userName }}님</p>
</template>

5. HTML 포함 텍스트 규칙

  • HTML이 포함된 텍스트는 v-html 디렉티브 사용
  • XSS 위험이 있으므로 신뢰할 수 있는 콘텐츠에만 사용
<!-- 올바른 예시 ✅ -->
<template>
  <p v-html="$t('terms.agreement')"></p>
</template>

<!-- 잘못된 예시 ❌ -->
<template>
  <p>본 <strong>서비스 약관</strong>에 동의합니다.</p>
</template>

6. 날짜/시간 형식화 규칙

  • 날짜와 시간은 $d 함수 사용
<!-- 올바른 예시 ✅ -->
<template>
  <p>{{ $d(new Date(), 'long') }}</p>
</template>

<!-- 잘못된 예시 ❌ -->
<template>
  <p>{{ new Date().toLocaleDateString() }}</p>
</template>
