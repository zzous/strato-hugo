컴포넌트 사용 규칙

  • 컴포넌트는 @strato/components 라이브러리를 기본으로 사용할 것
  • @strato/components 라이브러리에서 지원되지 않는 컴포넌트는 vuetify 라이브러리를 사용할 것

@strato/components 컴포넌트 사용 규칙

SCardTable 사용시

  • 페이지네이션이 필요한 경우 itemPerPage prop 사용
  • 검색 기능 사용시 smartSearch prop 활용
  • 테이블 행 높이는 density prop으로 조절 (default | comfortable | compact)

SModal 사용시

  • 모달 크기는 size prop으로 지정 (small | medium | large | x-large)
  • 헤더가 필요없는 경우 hideHeader prop 사용
  • 커스텀 버튼은 footer slot 활용

SFormTable & SFormItem 사용시

  • 필수 입력 항목은 required prop 추가
  • Vuetify 입력 컴포넌트와 함께 사용시 density="compact"와 hide-details="auto" 속성 추가
  • 입력 필드 우측 추가 요소는 outer-append slot 활용

SBtn 사용시

  • 색상은 blue, red, black 중 선택
  • 스타일 변형은 elevated 또는 outlined 사용
  • 아이콘 버튼의 경우 icon prop 활용

SScriptEditor 사용시

  • 언어별 구문 강조를 위해 lang prop 설정 (javascript, yaml 등)
  • 읽기 전용은 readonly prop 사용
  • 높이 조절이 필요한 경우 height prop 활용

SEmpty 사용시

  • 빈 상태 표시시 title과 description prop 활용
  • 커스텀 내용은 default slot 사용
  • 카드 스타일링은 elevation과 outlined prop 활용

SGlobalProgress 사용시

  • 전역 로딩 상태는 modelValue로 제어
  • 로딩 메시지는 loadingMessage prop으로 설정
  • teleportId는 기본값 "destination" 사용

SAccordionMenu 사용시

  • 제목은 title prop으로 설정 (필수값)
  • 제목 굵기 조절은 titleBold prop 사용
  • 내부 콘텐츠는 default slot 활용

SAlert 사용시

  • 알림 내용은 titleName prop으로 설정
  • 알림 표시 시간은 timeout prop으로 조절 (기본값 5000ms)
  • 닫기 버튼 표시 여부는 isClose prop으로 제어

SAvatar 사용시

  • 사용자 정보는 user prop으로 전달
  • 메뉴 아이템은 menuItems prop으로 설정
  • 뱃지는 badge slot 활용

SImg 사용시

  • 이미지 경로는 src 또는 srcUrl prop 사용 (둘 중 하나만 사용)
  • 로딩 중 표시할 이미지는 lazySrc prop 설정
  • public 폴더 이미지는 publicPath prop 활용

SFileInputBtn 사용시

  • 허용 파일 타입은 accept prop으로 지정
  • 버튼 텍스트는 buttonText prop으로 설정
  • 파일 선택 시 v-model로 파일 객체 바인딩

SShellScriptInput 사용시

  • 스크립트 내용은 v-model로 바인딩 (필수)
  • 입력 필드 라벨은 labelTitle prop으로 설정
  • 자동 라인 번호 표시 기능 제공

SHeader 사용시

  • 로그인 상태는 isLoggedIn prop으로 제어
  • 사용자 정보는 userInfo prop으로 전달
  • 서비스 메뉴는 serviceMenus prop으로 설정

SNavi 사용시

  • 서비스 이름은 serviceName prop으로 설정
  • 현재 메뉴 경로는 menuPath prop으로 전달
  • 메뉴 클릭 이벤트는 @click:menu로 처리

SProgressCircular 사용시

  • 크기는 size prop으로 조절 (기본값 50px)
  • 선 두께는 width prop으로 설정 (기본값 6px)
  • 자동 회전 애니메이션과 그라데이션 효과 제공

SSubHeader 사용시

  • 제목은 title prop으로 설정 (필수값)
  • 목록 개수 표시가 필요한 경우 showCnt prop 활용 (기본값 true)
  • SDataTable 또는 SCardTable과 함께 사용시:
    • showCnt를 true로 설정
    • listCnt prop에 테이블의 전체 아이템 개수 전달 필요
  • 커스텀 스타일링은 className prop 활용
  • 테이블과 함께 사용할 때는 반드시 테이블 상단에 배치할 것
  • 페이지 타이틀이 필요한 모든 페이지에서 일관되게 사용할 것

SDataTable 사용시

  • 데이터 목록은 items prop으로 전달 (필수값)
  • 컬럼 설정은 headers prop으로 정의 (필수값)
    • headers의 각 객체는 반드시 title과 key 키를 포함해야 함
    • title: 컬럼 헤더에 표시될 텍스트
    • key: items 배열의 객체에서 매칭될 데이터의 키값
  • 페이지네이션이 필요한 경우:
    • itemsPerPage prop으로 페이지당 아이템 수 설정
    • showFirstLastPage prop으로 첫/마지막 페이지 버튼 표시
    • totalItems prop으로 전체 아이템 수 전달
  • 정렬이 필요한 경우:
    • sortBy prop으로 정렬 기준 컬럼 지정
    • sortDesc prop으로 정렬 방향 설정
    • multiSort prop으로 다중 정렬 활성화
  • 선택 기능이 필요한 경우:
    • showSelect prop을 true로 설정
    • selectable prop으로 선택 가능한 아이템 제어
    • @select 이벤트로 선택된 아이템 처리
  • 로딩 상태 표시가 필요한 경우 loading prop 사용
  • 커스텀 셀 렌더링이 필요한 경우 template v-slot 활용
  • 대용량 데이터 처리시 virtual-scroll prop 활성화
  • 검색 기능이 필요한 경우 search prop과 searchFields prop 활용
  • 테이블 높이 조절이 필요한 경우 height prop 사용
  • 테이블 밀도 조절은 density prop 사용 (default | comfortable | compact)
  • 행 클릭 이벤트는 @click:row로 처리
  • 빈 데이터 표시는 no-data-text prop으로 커스터마이징
  • SDataTable의 slot 사용시 #[`item.{key}`] 형식을 사용할 것
    • 예: #[`item.name`], #[`item.status`]
    • 이 형식은 eslint 규칙과 호환되며, 코드의 일관성을 유지
    • 잘못된 예: #item.name, #item.status
    • 올바른 예: #[`item.name`], #[`item.status`]
  • Slot 내부에서 item 접근시 item.raw 대신 item을 직접 사용할 것
    • 잘못된 예: item.raw.name
    • 올바른 예: item.name
  • Slot 사용시 동적 속성 바인딩(:)을 사용할 것
    • 잘못된 예: v-slot:item.name
    • 올바른 예: #[`item.name`]

예시 코드

const headers = [
  { title: '이름', key: 'name' },
  { title: '나이', key: 'age' }
